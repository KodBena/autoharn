seen-red/apparatus-flip — banked RED evidence (both-polarity proof, fixture-census)

Tracker item "apparatus-flip-witnessing", from the "Unwitnessed apparatus flips" finding of the
NIST-lens re-litigation panel (design/MAINT-RELITIGATION-SYNTHESIS.md, 2026-07-12): "a governed
agent can Write .claude/apparatus.json and flip a mechanism to off mid-session with NO witnessed
event -- no refusal, no journal line, no ledger row, in a system whose evidentiary basis is 'hooks
only'". hooks/posttooluse_apparatus_flip.py closes this: before it existed, editing
.claude/apparatus.json produced no signal anywhere -- no PreToolUse hook governs that file (it is
not *.py, so hooks/pretooluse_change_gate.py's default governed patterns never match it), and no
PostToolUse hook watched it either (hooks/posttooluse_mutation_observer.py only attaches to the
Bash tool). A Write, Edit, or Bash-driven rewrite of apparatus.json was, and without this hook
still would be, silently invisible -- exactly the panel finding.

RED #0 (an out-of-frame hack-rationalization audit caught a SECOND red shape in this hook's own
FIRST version, 2026-07-12, before this commission was reported done -- CLAUDE.md's engineering-
responsibility corollary in action, not a corner routed around): the original "first observation
establishes the baseline silently" logic trusted whatever apparatus.json ALREADY contained at the
hook's very first-ever run for a world, so a flip that occurred before this hook had ever run once
(the single highest-value blind spot for the exact threat this hook exists to close) was itself
silently absorbed as trusted, with no event. Closed by comparing first observation against
`bootstrap/templates/apparatus.json` (the shipped scaffold default) rather than trusting it
blindly; see cases h/i below and the module docstring's BASELINE section for the residual gap that
remains (a world whose scaffold default is unreachable, or whose day-zero divergence happens to
restore the exact default bytes).
