   FIRST live run of this fixture was RED for a real reason, not a fixture defect --
   `serving/boundary_service.py`'s original `_query_json` raised

     RuntimeError: psql query returned no output where exactly one JSON value was expected

   on GET /health against WORLD PRE (a world with no `boundary-service` principal registered)
   and on the W7 allowed-leg's throwaway world, because `psql -tAq` prints the EMPTY STRING

   result. Fixed by making zero-output-lines map to Python `None` (the genuinely absent-value
   case every caller already handled: `row_by_id`'s 404, `service_principal_name`'s "not yet
   registered") rather than raising -- see the fix's own comment in `_query_json`. The GREEN
