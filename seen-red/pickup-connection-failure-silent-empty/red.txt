THE DEFECT: exit code 0, empty process stderr, ten "ERROR:" lines buried inside stdout
indistinguishable at a glance from the same script's own honest empty-result convention
("(no rows)", "(no resource declarations on record)", etc.) -- a hydration brief a caller
checking the exit code (the resumption doctrine's own entry point) cannot tell apart from an
honestly-empty tracker. Same defect class as verify-chain-error-conflation (fixed 2026-07-13,
commit 8768a01): a connection-level failure silently read and reported as if it were a
query-level answer.

THE FIX (this tracker item): `check_connectivity()` probes once, up front, with `SELECT 1;`
under `SET ROLE dep.role`; on failure `main()` prints a typed CANNOT-HYDRATE banner to STDERR
naming the psql error verbatim, prints NOTHING to stdout, and exits 3. See
seen-red/pickup-connection-failure-silent-empty/run_fixtures.py for the both-polarity live
proof (4/4 cases pass against real infra).
