# Report 01 — choco/ADR-0011-mechanization (chocofarm's "Mechanization Discipline" tenet — the harness north star)

## Summary

ADR-0011 is chocofarm's "convert every lapse to a mechanism, not more prose" tenet, structured as 4 rules with a closed enforcement-surface vocabulary that is the exact lexicon the harness's logic-gate layer should adopt. Its core operating principle is the harness's reason to exist: prose disciplines decay because they are policed by one person's memory, so the only durable corrective is a queryable/mechanical net that quantifies over the CLASS of a defect, not the instance. Two 2026-06-24 amendments already prototype the harness's two pillars in code — code_stamp.py (CAPABILITY/provenance: every benchmark reading auto-stamps git commit+tree state) and exp_db.py (ACCOUNTABILITY ledger: append-only Postgres findings linking reading→interpretation→commit with supersede chains and a pre-registration layer). ADR-0000 makes ownership explicit and load-bearing for the harness: question (b) "what operational lapse let this happen" is aimed at the EXECUTIVE not the implementer — "when ADR-0011 is violated it is structurally the executive's to own, because the executive owns the enforcement-surface." The net/shape/integrity trio framing (0011=net, 0012=shape, 0013=integrity, rooted in 0000) is the four-axis schema a harness "ADR-0011 safety net" would enforce per defect. The central GAP: ADR-0011 itself is self-declared review-only with "no automated enforcement of this tenet itself" and the architectural audit as a run-on-demand absence-detector — that absent CI sweep IS the harness.

## Key facts

- ADR-0011's Decision is FOUR rules (0011 lines 58-137): Rule 1 'disciplines declare their enforcement surface' against a CLOSED vocabulary; Rule 2 'recurrence converts to mechanism, not more prose'; Rule 3 'mechanisms adopt measure-first'; Rule 4 'nets quantify over the class, not the instance.'
- Rule 1's closed enforcement-surface vocabulary (0011 lines 66-75) is the exact lexicon for the harness logic-gate layer: construction/import-time | test/CI gate | write-time data constraint | run-time invariant | review-only. 'Review-only is legitimate but presumptively decaying — declaring it makes that a visible, challengeable choice' (lines 77-78).
- Rule 4 (0011 lines 116-137) is the formal spec for what a 'net' must be: it 'keys on a structural slot, a name/shape predicate, or a derived-from-one-source invariant' — NEVER an enumeration of instances, which 'fail open at the next instance.' The anti-example is the `it + 1_000_000` magic disambiguator that breaks at iters>=1e6 (fix: namespace by (run,phase,version), R14).
- OWNERSHIP rule (the key accountability fact, ADR-0000 Rule 2(b), lines 207-217): question (b) 'what operational lapse let this happen' reads operational as EXECUTIVE — 'aimed at the maintainer/ratifier, NOT the implementer.' Verbatim: 'When ADR-0011 is violated it is structurally the executive's to own, because the executive owns the enforcement-surface — a recurrence that was never mechanized is a guard the maintainer did not build, not an implementer who erred.'
- The net/shape/integrity trio (ADR-0000 Genre, lines 15-22): ADR-0000 is the root the trio descends from — ADR-0012 is its *shape* (types/illegal-states-unrepresentable), ADR-0011 is its *operational net* (mechanism), ADR-0013 is its *integrity* (real fix to ratified end). ADR-0014 is the escape hatch when the reflex stalls. A complete defect disposition carries all three facets (ADR-0000 Rule 3, lines 240-256).
- The WORKED 'lapse converted to CI/mechanism' examples ADR-0011 cites: (1) `FeatureLayout` (az/features.py) — three-writer feature-layout triplication converted to ONE ordered block table read by name with a fail-loud contiguous-partition assertion that blocks cover [0,dim) (0011 lines 47-54, 121-126); (2) hp-registry strict `decode_config` + `hp/schema.py` `check_invariants` write-time constraint (lines 68-72); (3) the jax/numpy equivalence test at ABS_TOL=1e-4 (test/CI gate, measure-first per Rule 3).
- Amendment 1 (2026-06-24, the commit-stamp net, 0011 lines 197-230) is a prototype of the harness CAPABILITY/provenance pillar: a '+31%' throughput win 'could not be pinned to a commit' and failed to reproduce — 'an attributed reading with no record of the code that produced it is unattributable by construction.' Mechanism: throughput-lab/harness/code_stamp.py emits git short-hash + tree state (clean|DIRTY) on EVERY reading (the class), DIRTY marking a non-reproducible artifact.
- Amendment 2 (2026-06-24, the Witness chain, 0011 lines 232-261) is a prototype of the harness ACCOUNTABILITY/ledger pillar: interpretations move into `tlab_finding` (throughput-lab/harness/exp_db.py), an append-only Postgres table SEPARATE from `tlab_reading` measurements so 'a reading-of the data recorded as the data' is 'structurally unrepresentable.' Findings carry motivation+interpretation, status in {provisional,confirmed,retracted}, the commit the belief formed against, and a `supersedes` link (the Witness->Correction chain, never rewritten).
- ADR-0011's OWN Rule-1 declaration is review-only (0011 Self-application, lines 138-149): 'chocofarm has no CI sweep that detects a discipline-stating rule lacking an enforcement declaration; the architectural audit is that detector, run on demand.' Consequences/Negative confirms: 'chocofarm has no automated enforcement of this tenet itself; it is review-and-audit-policed' (lines 161-163).
- Rule 3 measure-first (0011 lines 102-114): 'A check at full strength lands only on a zero-or-fully-triaged baseline'; ABS_TOL=1e-4 'was chosen comfortably above the observed float error, a measured threshold, not a guessed one.' Guards against cargo-cult mechanisms — 'a check at full strength on an un-triaged baseline is worse than none' (line 162).

## Gaps (where the haphazardness lives)

- ADR-0011 has NO automated enforcement of itself: 'chocofarm has no CI sweep that detects a discipline-stating rule lacking an enforcement declaration; the architectural audit is that detector, run on demand' (Self-application, lines 145-146) and 'chocofarm has no automated enforcement of this tenet itself; it is review-and-audit-policed' (lines 162-163). The missing sweep IS the harness's core deliverable.
- Rule 1 declarations are unstructured prose scattered across ADRs (e.g. ADR-0000 Rule rubrics, ADR-0012 P8/P9, ADR-0013) — there is no machine-readable registry mapping {discipline-rule -> enforcement-surface -> mechanism file:line}, so 'does every rule declare a surface, and does the named mechanism still exist' cannot be queried. ADR-0011 Revisit #3 even names 'the enforcement-surface declarations are the transfer manifest' but no manifest is materialized.
- The accountability ledger (exp_db.py) links reading->interpretation->commit but has NO link to the Claude Code SESSION that produced a reading or change (grep confirms no session_id/agent column; 'single-session' appears only as prose in CHECK comments). The maintainer's stated gap — 'no link from a bug to the session that introduced it' — is exactly this missing column.
- code_stamp's DIRTY state is recorded but not gated: a DIRTY reading is marked 'provisional' by convention only; nothing prevents a DIRTY (non-reproducible) number from being banked as a finding or motivating a code change. A logic-gate could refuse promoting a finding to status='confirmed' while its underlying readings are DIRTY.
- Rule 3 'measure-first / baseline triaged before full-strength check' is review-policed — there is no mechanism that records a check's adopted baseline or blocks a full-strength assertion landing on an un-triaged baseline (the cargo-cult-mechanism risk ADR-0011 lines 159-162 names).
- The net/shape/integrity completeness check (ADR-0000 Rule 3: a defect disposition must carry all three facets) is purely a reviewer reading a fix against three facets — no gate verifies a defect-fix PR names its foreclosed class (shape), its mechanism (net), and its verification (integrity). ADR-0000 Revisit #3 explicitly anticipates 'a review-checklist that a defect-fix PR names its foreclosed class' as a future mechanism not yet built.
- The pre-registration layer (tlab_prereg) exists in the throughput-lab toy but is not generalized: only the benchmark domain has hypothesis<->criterion<->verdict separation; the broader 'attributable link between git-hash, benchmark, hypothesis, interpretation, session' the harness wants is not a cross-project schema.

## Harness hooks (where a registry / ledger / logic-gate plugs in)

- A CAPABILITY/enforcement registry as a logic program over Rule 1's closed vocabulary: ingest every ADR rule (and CLAUDE.md convention / docstring contract) as a fact {rule_id, declared_surface in {construction,ci_gate,write_constraint,runtime_invariant,review_only}, mechanism_path, mechanism_symbol}; a Datalog/Z3 gate then proves (a) every discipline-stating rule declares a surface (the missing CI sweep, lines 145-146), and (b) every named mechanism path:symbol still resolves (ADR-0011 Revisit #2's doc-side resolution check, line 172).
- Generalize code_stamp.py into the harness's universal provenance-stamp primitive: any tool/agent emitting a result calls it; the {commit, tree} becomes a NOT-NULL attribution column everywhere (it is already the pattern in tlab_reading lines 200-201 and tlab_finding lines 244-245).
- Extend exp_db.py's tlab_reading/tlab_finding with a `session_id` (Claude Code session) column to close the maintainer's named gap (bug<->introducing-session); the supersedes self-FK + motivated_change column already model the interpretation->code-change loop, so adding session turns it into the full git-hash<->benchmark<->hypothesis<->interpretation<->session ledger.
- A Rule-4 'net-shape' linter/logic-gate: given a proposed mechanism, prove it keys on a class predicate (structural slot / name-shape / derived-from-one-source) rather than an enumeration — formalizing the FeatureLayout-vs-`it+1_000_000` distinction (lines 121-137) as a checkable property (a clear Z3/OR-tools use the maintainer wants to surface).
- A net/shape/integrity completeness gate on defect-fix PRs (ADR-0000 Rule 3 + Revisit #3): require a fix to declare its foreclosed class (shape/0012), its recurrence-mechanism + enforcement surface (net/0011), and its artifact verification (integrity/0013) — a three-slot logic gate that fails open if any facet is unfilled, the programmatic form of the reviewer's three-facet read.
- An ownership-attribution gate encoding ADR-0000 Rule 2(b): when a recurrence is detected (same defect class twice), the gate attributes the lapse to the EXECUTIVE/enforcement-surface owner (the un-built guard), not the implementer — making 'who owns this mechanization failure' a query, not a judgment call (lines 207-217).
- Promote tlab_prereg (immutable criterion + separate conclusion) into a cross-project pre-registration table so any hypothesis gets a falsifiable, pre-declared verdict bin before data exists — the structural anti-rationalization mechanism (exp_db.py lines 277-290) the harness can reuse to make 'was this result decisive?' mechanical rather than rhetorical.
- A DIRTY-state promotion gate: a logic rule refusing status='confirmed' on any finding whose supporting readings have git_tree='DIRTY', operationalizing 'a DIRTY tree marks a non-reproducible artifact... provisional until committed' (code_stamp.py docstring lines 8-10).
