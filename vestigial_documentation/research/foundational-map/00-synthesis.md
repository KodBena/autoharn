## Headline

claude_harness IS the CI sweep ADR-0011 openly declares it lacks ("chocofarm has no automated enforcement of this tenet itself; it is review-and-audit-policed", chocofarm 0011:161-163; omega's only nets are a pre-merge-checklist §H line + two audit-sweep absence-checks, omega 0011:130-145). It is not new structure: it is omega's work-status Postgres anti-corruption layer (`tools/work-status/schema.sql` — relational SSOT + `work_status_violations` query-time gate + `audit_log`/`table_asof` provenance) generalized from ONE domain (todo items) to the two named haphazard surfaces — a CAPABILITY registry the agent PULLS at point-of-need (replacing the PUSH of 37 stale-injected memories + the gitignored prose SSOT `services_local.gitignore`) and a PROVENANCE ledger that already exists in skeletal, fork-divergent form (chocofarm's `exp_db.py` measurement⊥interpretation store + `code_stamp.py`; omega's `audit_log` actor+commit_sha). The spine that unifies them is the net/shape/integrity trio rooted in ADR-0000 (0012=shape/illegal-states-unrepresentable, 0011=operational net/mechanism, 0013=integrity/real-fix): a LOGIC safety net of per-store `*_violations` views backed by already-installed engines (Z3 4.16.0 + OR-Tools CP-SAT 9.15 in `/home/bork/w/vdc/venvs/generic`, Postgres `WITH RECURSIVE`) that quantify over the CLASS of a defect per Rule 4, never an enumeration. By ADR-0000 Rule 2(b) the harness is structurally the executive's to own — "a recurrence that was never mechanized is a guard the maintainer did not build" — so the harness's first deliverable is the meta-sweep that proves every discipline-stating rule declares an enforcement surface and its named mechanism still resolves.

## Pillar 1 — Capability Registry (intent SSOT)

**Problem.** Resource/capability facts are PUSH, not PULL. The authoritative inventory is a gitignored prose file (omega/services_local.gitignore: postgres hosts at 192.168.122.1, redis topology @6379/6380/8192, tensorboard @192.168.122.68:6006, venvs, the per-session-volatile X11 cookie host that MUST be derived at runtime) plus 37 passively-injected Claude memories indexed by per-project MEMORY.md. The agent leans on memory hardest exactly when context is thinnest — the opposite of intent. There is no liveness check and no freshness stamp: a refuted belief (e.g. the 'core-3 pinning benefit', refuted in-place in host-cpu-isolation-pinning.md and echoed in topology_enum.py's own CAVEAT) persists until a human rewrites it. Most consequentially for the brief: the OR/automated-reasoning tools (Z3 4.16.0, OR-Tools 9.15 CP-SAT, both actually installed in /home/bork/w/vdc/venvs/generic) are enumerated NOWHERE, so agents default to the StackExchange-shaped statistical path instead of the provable one. The same migration the maintainer already executed once (hand-edited work-status.json → queried SQL with invariant gate + audit) is the exact antidote.

**Design.** A queried SQL store on the existing host (192.168.122.1), one row per capability, that the agent PULLS at point-of-need (SELECT addr, probe_cmd FROM capability WHERE name=...). It reuses the anti-corruption pattern wholesale: closed-vocabulary CHECK enums (ADR-0008), a capability_violations VIEW (omega's empty⇒clean gate), and an executable liveness probe per row so the standing 'assume up but surface a failed check loudly' rule (services_local:12-15, ADR-0002) becomes a queried fact, not cached advice. Volatile facts are stored as a derive_cmd (a command to run), NEVER a literal — the X11 cookie host is the canonical example. Each row carries the measurement⊥interpretation discipline (validated_at + status provisional|confirmed|retracted + git_commit/session_id) so a refuted resource belief is SUPERSEDED queryably, not silently stale. Z3, OR-Tools, the Criterion partition-validator, the perf/bench harness (bench_hotpath, bench_equivalence, test_jax_equivalence, perf-capture.mjs, the window.__perfScenario set) are first-class rows with invoke_example + probe so agents discover provable tools by query.

**Schema sketch.**

capability(cap_id PK, name text UNIQUE, kind text CHECK IN ('service','db','venv','solver','tool','path','display'), addr text, invoke_example text, probe_cmd text NOT NULL, derive_cmd text NULL, owning_memory_id text, git_commit text, session_id text, validated_at timestamptz, status text CHECK IN ('provisional','confirmed','retracted'), supersedes bigint REFERENCES capability(cap_id), notes text). capability_probe(probe_id PK, cap_id FK, ran_at timestamptz, ok bool, detail text)  -- liveness history = the measurement layer. Seed rows: ('z3','solver','/home/bork/w/vdc/venvs/generic', invoke='from z3 import Solver', probe='/home/bork/w/vdc/venvs/generic/bin/python -c "import z3"'); ('ortools-cpsat','solver',same venv, probe='...python -c "from ortools.sat.python import cp_model"'); ('research_redis','service','127.0.0.1:6380', derive=null, probe='redis-cli -p 6380 ping'); ('x11_headed','display',':0', derive='xauth -f ~/.Xauthority list | awk ...', probe='xset q'). VIEW capability_violations: a status='confirmed' row whose latest capability_probe.ok IS NOT TRUE; a kind='solver' row whose probe fails (registered-but-absent); any row whose value duplicates a derive_cmd-only fact stored as a literal (the X11 anti-pattern).

**Reuses.**

- omega work-status anti-corruption pattern verbatim: CHECK-enum write-time constraints + capability_violations VIEW + the empty⇒clean CI runner (schema.sql:45-128)
- migrate-to-pg.py self-certify → the registry's liveness self-certify: a tool that pings each service / imports each venv package and fails loudly if a registered capability is absent (migrate-to-pg.py:120-162)
- exp_db.py measurement⊥interpretation + supersedes: capability_probe is the reading, the capability row is the (supersedable) interpretation (exp_db.py:191-266)
- services_local.gitignore as the seed source of truth (one-time migration, same shape as work-status.json → SQL)

**Risks.**

- services_local.gitignore is gitignored precisely because its facts are machine-local and go stale off this box; the registry store must ALSO stay machine-local (it lives on 192.168.122.1 already) — do not let migration leak addresses/credentials into a tracked artifact
- a probe_cmd is itself code that can rot; the registry must self-certify its own probes (the meta-sweep eats its own dog food) or it recreates the stale-belief failure one level up
- per-row probe queries over the .1 latency hop risk N+1; bundle probes (services_local already warns about per-position fetch-bundling)
- deciding kind's closed vocabulary too narrowly re-introduces the ADR-0008 fuzzy-match defect (forcing a capability into the closest wrong bucket); leave an honest 'extra' jsonb tail as exp_db/work-status both do


## Pillar 2 — Provenance / Accountability Ledger

**Problem.** A perf claim is hand-typed prose in a commit body ('11.6x faster', 'RTT 453.5→335.8us (-26%)', 'saturated 4281→7776 msg/s (+85%)' on chocofarm's encode_response commit; omega's b3bfe8c1 '17.1s→8.3s (-51%)') machine-linked to NOTHING — not the reading rows, not the environment, not the finding it tested, not the Claude session that authored it, and nothing checks the number equals median(candidate)/median(baseline). The one link that exists is BACKWARDS: tlab_reading.git_commit is the commit the reading was MEASURED at (usually a DIRTY candidate or the baseline), so the perf(...) BANKING commit's hash appears in zero reading rows. Findings are cited as free text ('finding #32', 'tlab_finding #37') — human pointers, not FKs. There is NO session column anywhere (chocofarm has only the model-name trailer 'Co-Authored-By: Claude Opus 4.8'; omega has only actor=application_name defaulting to 'unknown'/'coordinator'), so the maintainer's stated gap — bug → introducing session — is unanswerable. omega's capture path is worse: perf-capture.mjs writes traces named ONLY <scenario>-<ISO-timestamp>.json (line ~115), throwing away the in-process cfg object (model/visits/adaptive/boards) and the env mode (headless/headed/--connect), and never touches git.

**Design.** Adopt chocofarm's exp_db.py as the shared ledger and close exactly four gaps. (1) SESSION: a sessions table + a session_id column on every provenance-stamped row (tlab_reading/finding/prereg), sourced from a Claude Code SessionStart hook that exports an env var / sets the Postgres GUC, so code_stamp() emits {commit, tree, session_id} as one unit and omega's audit_log.actor stops being free-text. (2) BANKING EDGE: a banked_commit column (the perf(...) commit that ASSERTS a win, the inverse of the measured-at git_commit) so 'which commit asserts this and on what evidence' is a join. (3) TRAILER→FK: a git commit-message-trailer parser (Tlab-Reading: <ids>, Tlab-Finding: <id>, Tlab-Prereg: <slug>, Claude-Session: <uuid>) run as a build/CI gate that turns today's 'finding #32' prose into provenance_edge rows. (4) ENV + ARTIFACT: a content-hash tlab_env table (mirroring tlab_config dedupe) and a tlab_artifact table (path + sha256 + kind) so 'intra-VM', '__AVX2__', 'taskset -c 3', the ~/w/vdc/chocobo/profiles/*.perf.data and omega's ~/w/vdc/chromium_profiles/*.json become code-addressable, not prose. Port omega's audit_log triggers + table_asof/asof.sh onto the tlab_* tables to give chocofarm per-commit time-travel and the bug→introducing-session trace it lacks. At omega's capture write-site (perf-capture.mjs ~196-200) emit a sidecar reading row (git rev-parse HEAD+dirty, the full cfg, the env mode, session_id).

**Schema sketch.**

ALTER exp_db: tlab_reading ADD session_id text, ADD banked_commit text NULL, ADD env_id bigint REFERENCES tlab_env; tlab_finding ADD session_id text; tlab_prereg ADD session_id text. NEW session(session_id text PK, model text, agent text, project text, started_at timestamptz, ended_at timestamptz). NEW tlab_env(env_id bigserial PK, env_key text UNIQUE /*content hash*/, cpu_model text, governor text, taskset text, kernel text, build_flags jsonb /*AVX2*/, cuda_ver text, jax_ver text, vm_boundary text). NEW tlab_artifact(artifact_id bigserial PK, reading_id bigint REFERENCES tlab_reading, path text NOT NULL, sha256 text NOT NULL, kind text CHECK IN ('bench_log','tensorboard_run','perf_data')). NEW provenance_edge(edge_id bigserial PK, from_commit text NOT NULL, kind text CHECK IN ('Tlab-Reading','Tlab-Finding','Tlab-Prereg','Claude-Session'), to_reading_id bigint NULL REFERENCES tlab_reading, to_finding_id bigint NULL REFERENCES tlab_finding). Port audit_log + record_audit() triggers + table_asof() (omega schema.sql:159-213) onto tlab_*. VIEW provenance_violations (modeled on work_status_violations 'shipped-without-ship-ref'): (a) a tlab_finding status='confirmed' whose supporting readings include any git_tree='DIRTY'; (b) a banked_commit asserting a %/x token with no provenance_edge to a baseline-reading AND a candidate-reading; (c) a result-bearing reading with session_id IS NULL; (d) a tlab_finding.refs that cites no resolvable reading_id (BACKLOG's policy 'a finding cites a reading_id not a bare figure', ~line 207, made a write-time constraint).

**Reuses.**

- exp_db.py tlab_reading/finding/prereg whole schema incl. supersedes chain, status enum, criterion partition (exp_db.py:191-340)
- code_stamp.py as the {commit,tree[,session_id]} emitter (code_stamp.py)
- omega audit_log + record_audit() + table_asof()/asof.sh ported onto tlab_* (omega schema.sql:159-213)
- work_status_violations 'shipped-without-ship-ref' as the literal template for the 'banked-perf-claim-without-evidence-reading' gate (omega schema.sql:120-128)
- tlab_config content-hash dedupe as the pattern for tlab_env (exp_db.py:167-189)

**Risks.**

- session_id provenance depends on a SessionStart hook firing reliably; a missing session_id must be honest-NULL (the new 'DIRTY'), never fabricated — ADR-0002 fail-loud, ADR-0008 negative register (refuse to invent a category)
- the perf-ratio gate (claimed == median(candidate)/median(baseline)) needs a declared tolerance and a declared 'which readings are baseline vs candidate' — without prereg this re-opens the retro-fit narrative the prereg layer exists to close; gate only where the cohort is unambiguous
- banked_commit can only be written AFTER the commit exists, but the reading is recorded BEFORE — needs a post-commit reconciliation pass (the trailer parser), which is itself a mechanism that can lag
- two disjoint stores today (chocofarm tlab_* with no audit; omega audit_log with no readings) — unifying them risks breaking each project's self-certify round-trip unless migrate-to-pg's faithful-image contract is extended to cover the merged schema
- DIRTY-blocks-confirmation: choosing write-time hard-block vs query-time flag is a Rule-1 surface decision with real friction cost (a legitimate provisional finding on a dirty tree is common mid-session)


## Pillar 3 — Logic Safety Net (the ADR-0011 spine)

**Problem.** ADR-0011 is self-admittedly un-mechanized at the meta level: 'chocofarm has no CI sweep that detects a discipline-stating rule lacking an enforcement declaration; the architectural audit is that detector, run on demand' (chocofarm 0011:145-146), and omega's only nets are a checklist line + two audit-sweep absence-checks (omega 0011:130-145) — all human-attention-gated. Below that meta gap, the actual invariants are hand-written imperative scans with magic-literal baselines (band=30, danglers=38, cycles=0/0), inline CHECK strings scattered per-file, and prose supersession ('Amended 2026-06-15…', ADR-0000 'Provisional (emergency)'). The 'provable beats statistical hunch' capability is absent from the gate layer even though band-ordering, acyclicity, partition-coverage, and equivalence are all DECIDABLE. The safety net IS the absent CI sweep, expressed as per-store *_violations views backed by real engines.

**Classical uses (provable invariants).**

- Z3 (installed, generic venv) — PARTITION VALIDATOR: lift the Criterion dataclass (no gaps/no overlaps over a metric value-line) and FeatureLayout's contiguous-cover-[0,dim) assertion into a reusable interval-coverage proof; provable, the flagship 'provable beats hunch' instance (chocofarm 0011:121-126; exp_db tlab_prereg criterion)
- Z3 — TWO-TIER EQUIVALENCE TYPE-CHECK: kind=logic_invariant ⇒ assertion must be ==0.0 (a tolerance on a logic invariant is unsatisfiable); kind=numeric ⇒ must be statistical-within-MC-CI with reported standard error, N>=300/>=2 seeds. A category error in either direction (pinning a float-sensitive rate bit-exact; relaxing a logic invariant to a tolerance) is a gate failure (chocofarm ADR-0009:47-67)
- Z3 — PERF-RATIO CHECK: prove the claimed ratio == median(candidate)/median(baseline) within tolerance over the cited readings, replacing the hand-typed '11.6x' prose
- Postgres WITH RECURSIVE (installed, ALREADY the de-facto Datalog engine, work_status_violations) — the meta-sweep (every discipline-stating rule declares a surface; every named mechanism path:symbol still resolves), enum-coverage (every closed-vocabulary literal has a membership; every vocabulary element fits something), fossil/derivable-field detection (a stored field that is a function of live geometry — the instance.json overlaps/delta_treasures pattern, ADR-0008:56-69), cycle/ship-ref gates
- OR-Tools CP-SAT (installed + ALREADY USED at chocofarm hp/backends/cpsat.py) — combinatorial feasibility / topology enumeration with symmetry dedup, and any scheduling/feasibility obligation a violations view emits for an external checker to discharge and write provable pass/fail back as ledger rows

**Non-classical uses (defeasible / temporal / paraconsistent).**

- DEFEASIBLE / NON-MONOTONIC — generalize tlab_finding.supersedes (exp_db.py:256-258) into a corpus-wide append-only supersede graph covering ADR amendments, provisional decisions, and refuted resource beliefs (the 'core-3 pinning' refutation). 'Current belief on scope S = the node nothing supersedes' is the non-monotonic query; ADR-0000's Provisional status and ADR-0008's inline amendment are the first rows. The prior is NEVER rewritten (Witness→Correction).
- TEMPORAL / LTL-shaped — the prereg invariant (criterion.git_commit MUST precede the result's commit; one experiment, one immutable criterion, exp_db tlab_prereg) generalized to 'hypothesis declared-before / interpretation linked-after' for any session, and to ADR-0014's session-trace ordering (N consecutive edits citing the SAME diagnosis against the SAME unresolved symptom ⇒ emit a second-opinion trigger)
- PARACONSISTENT / 3-VALUED — represent the 'deliberately-imprecise', 'cited-not-rerun', STALE, 'detector-coupled-and-suspect' tags (ADR-0008:161-168) and git_tree='DIRTY' as an explicit third value 'unknown/suspect' the gate must NOT explode on — so conflicting advisories and not-yet-corroborated provisional findings coexist without forcing a true/false collapse

**Candidate engines.**

- z3-solver 4.16.0 — INSTALLED at /home/bork/w/vdc/venvs/generic (verified `import z3`); the partition/equivalence/ratio proofs
- OR-Tools 9.15.6755 CP-SAT — INSTALLED at /home/bork/w/vdc/venvs/generic (verified `from ortools.sat.python import cp_model`) and ALREADY in productive use (hp/backends/cpsat.py)
- PostgreSQL WITH RECURSIVE — INSTALLED and IN USE (work_status_violations CYCLE clause); the Datalog-shaped recursion + all set/coverage gates, no new dependency
- clingo / ASP (clasp) — NOT installed; the honest engine for genuine defeasible/non-monotonic reasoning if the supersede-graph + paraconsistent layer outgrows what append-only SQL + a 'current = nothing supersedes' view can express (install-required decision)
- Soufflé (Datalog) — NOT installed; only if recursive-SQL ergonomics become the bottleneck for the meta-sweep over the whole ADR corpus (install-required decision)

**ADR-0011 tie-in.** The safety net is literally the CI sweep ADR-0011 declares absent (chocofarm 0011:145-146,161-163; omega 0011:130-145). Each harness gate DECLARES its own enforcement surface against Rule 1's closed vocabulary (so the harness's own rules are not review-only), and the meta-sweep proves Rule 1 for the whole corpus: every discipline-stating rule names a surface and its mechanism resolves. It honors Rule 4 by keying every gate on a CLASS predicate (structural slot / name-shape / derived-from-one-source) not an enumeration — formalizing the FeatureLayout-vs-`it+1_000_000` distinction as a checkable property. It honors Rule 3 (measure-first): a new gate must record its adopted baseline before landing at full strength, because 'a check at full strength on an un-triaged baseline is worse than none' (0011:159-162). It completes the net/shape/integrity trio (ADR-0000 Rule 3): a defect-fix gate that fails open unless the fix names its foreclosed class (shape/0012), its mechanism+surface (net/0011), and its verification (integrity/0013).

**Risks.**

- the harness can itself cargo-cult Rule 3: landing a full-strength gate on an un-triaged baseline is worse than none — every new gate needs a measured baseline record first, and the harness must eat its own dog food (a meta-sweep over the harness's OWN rules)
- over-formalizing judgment-shaped findings (e.g. BAND_EXCEPTIONS adjudications) as hard logic risks omega Rule 5's 'bungled compliance, strictly worse than missing compliance' (template-not-tollgate) — keep judgment surfaces advisory
- the non-classical layer is the least-supported by installed engines: defeasible/temporal/paraconsistent are prototyped only as append-only SQL semantics; promoting them to clingo/Soufflé adds a dependency the capability registry must then itself track and probe
- a paraconsistent 3-valued layer that is too permissive lets contradictions persist quietly — the value of the DIRTY/suspect tag is that it BLOCKS promotion-to-confirmed, not that it merely records
- Z3/CP-SAT proofs are only as sound as the model extraction; a buggy SQL→SMT translation yields a confidently-wrong 'provable' result, the exact false-authority the brief warns agents fall for


## Cross-project invariants (machine-checkable across BOTH projects)

- Every discipline-stating rule in BOTH ADR corpora declares an enforcement surface drawn from the ONE reconciled Rule-1 closed vocabulary (the fork — omega 7 rungs vs chocofarm 5 — is itself an ADR-0008 closed-vocabulary violation across the fork boundary; the harness hosts the single set both map onto)
- Every named mechanism (path:symbol) cited by a Rule-1 declaration still resolves on disk — the doc-side resolution check ADR-0011 Revisit #2 names, run centrally over both repos
- Every result-bearing write (benchmark reading, perf capture, ship-closure) carries a code_stamp {commit, tree} AND a session_id; a missing one is honest-NULL, never fabricated (ADR-0002 / ADR-0008 negative register)
- No finding/belief is promoted to status='confirmed' while any of its supporting readings has git_tree='DIRTY' (a DIRTY tree marks a non-reproducible artifact in both code_stamp.py and exp_db)
- Every perf-claim token ('faster'/'regression'/'-NN%'/'NNx'/'equivalent'/'matches the baseline') in a commit/PR/worklog references a ledger reading_id (or carries an explicit '[unsubstantiated]'/loud marker) — the ADR-0009 Revisit #4 claim-token scanner, identical policy in both corpora
- Every closed-vocabulary enum value is mirrored to its single SSOT (the driver CHECK ↔ EnumSet pattern, exp_db.py:137-175) — no classification literal exists without a membership, and no vocabulary element fits nothing (ADR-0008 positive/negative registers)
- Every shipped/banked item has a resolvable ship-ref of kind commit/pr/worklog (omega's work_status_violations rule, generalized to chocofarm's banked-perf-commit ⇒ has evidence-reading)
- When the SAME defect class recurs twice, the lapse is attributed to the enforcement-surface OWNER (the executive who did not build the guard), not the implementer — ADR-0000 Rule 2(b) made a query, not a judgment call, over both corpora
