# Report 03 — choco / typing + classification-logic (ADR-0008 + ADR-0000) — the philosophical root for the harness LOGIC layer (classical + non-classical)

## Summary

ADR-0000 and ADR-0008 are the chocofarm corpus's explicit articulation of "make illegal states unrepresentable" as a first-move reflex, and "a vocabulary is honest only when it precisely fits the territory." Together they already frame defects as design signals to be foreclosed by TYPES (classical: refined types, CHECK enums, partition invariants) rather than patched, and frame classification as a closed-vocabulary fit problem (Datalog/ASP/SMT-shaped). The corpus has ALREADY mechanized a chunk of this — `throughput-lab/harness/exp_db.py` is a working provenance/accountability ledger (Postgres) where measurement ⊥ interpretation, findings carry a closed-vocabulary `status {provisional,confirmed,retracted}`, an append-only `supersedes` chain (defeasible/temporal logic in SQL), and pre-registration tables make a decisiveness criterion a MECHANICAL partition check. The gaps the harness should close: every discipline is honestly self-declared as "review-only, policy not mechanism" with a named mechanization trigger — these triggers ARE the logic-gate plug points. The classical invariants (enum-coverage, fossil-field, contiguous-partition, type-fit) belong in Datalog/SMT; the non-classical invariants (ADR supersession, provisional-until-corroborated, conflicting advisories, criterion-before-result temporal ordering) belong in defeasible/temporal logic and are already prototyped in the ledger's append-only supersede semantics.

## Key facts

- ADR-0000 Rule 2 mandates that on ANY identified defect the FIRST move is two prior questions before any fix: (a) 'what type would make this defect class unrepresentable?' and (b) 'what operational/executive lapse let it recur?'. A patch authored without asking them is ITSELF a defect (0000 lines 187-226).
- ADR-0000's four specimens are each a defect-class foreclosed by a TYPE, not a guard: BoundedBatch (boundary validator makes wrong shape/width/dtype unrepresentable, lines 74-89), CellLedger (a reconciliation type making cross-currency counter comparison 'a sentence you cannot write', lines 92-104 — explicitly an ADR-0008 category error fixed structurally), byte-budgeted high-water-mark (bound derived from one source, lines 106-117), bounded blocking queue (lines 119-127).
- ADR-0000 explicitly names the enforcement-surface CLOSED vocabulary it declares rules against: construction/import-time | test/CI gate | write-time data constraint | run-time invariant | review-only (lines 162-165; defined in ADR-0011 Rule 1 lines 67-75).
- ADR-0008 distinguishes two registers of category error: POSITIVE (refuse fuzzy match against an inadequate vocabulary — revise the vocabulary, don't pick closest fit) and NEGATIVE (refuse to fabricate a category under ambiguity — default to flat/named-incomplete). Both are closed-vocabulary fit problems (lines 79-115).
- ADR-0008 severity is calibrated by a SUBSTITUTION TEST: name the failure shape in its most general form, list all surfaces it could apply to, calibrate to the worst case — NOT the observed instance's cost (lines 118-130, Rule 4 line 147). This is quantify-over-the-class, the same logic as ADR-0011 Rule 4.
- ADR-0008's negative-register substrate is the 'fossil arrays' in instance.json — superseded 16-region overlaps/delta_treasures left standing where a reader can't tell live from dead; remediated by re-derivation from live geometry (lines 56-69). This is a derivable-field / stale-categorisation invariant a Datalog rule could enforce.
- ADR-0008 itself admits 'Discipline is policy, not mechanism... no automated check (ADR-0011 Rule 1: a declared review-only surface; an enum-coverage or fabricated-parent check would be the mechanization trigger)' (lines 194-197) and Revisit #4 names exactly: an enum-coverage check, a fabricated-parent (single-occupant synthetic directory) detector, a fossil-field check on instance.json (lines 214-217).
- ADR-0008 'Deliberately-imprecise tag' exception (lines 161-168) is an explicit NON-CLASSICAL construct: a tag that admits the classification is incomplete (analyzer's 'detector-coupled and therefore suspect', STALE marker, 'cited-not-rerun' evidence tag) is an explicit REFUSAL to classify until the case firms up — i.e. a paraconsistent/3-valued 'unknown' rather than a forced boolean.
- ADR-0000 is filed Status PROVISIONAL (emergency) and openly flags an unresolved ADR-0008 genre tension about whether 0000 is a distinct ADR or a meta-frame (lines 3-14, Revisit #1 lines 346-359) — a real-world instance of a PROVISIONAL / conflicting-classification decision the non-classical layer must represent rather than collapse.
- A working ledger ALREADY EXISTS: throughput-lab/harness/exp_db.py — Postgres, with tlab_reading (measurements, NOT-NULL git_commit + git_tree CHECK IN ('clean','DIRTY') provenance) STRICTLY SEPARATE from tlab_finding (authored interpretations). This is the (1) capability/provenance substrate the harness should generalize.

## Gaps (where the haphazardness lives)

- ADR-0008's entire discipline is self-declared 'policy, not mechanism' — there is NO automated enum-coverage check, NO fabricated-parent (single-occupant synthetic directory) detector, NO fossil-field check on instance.json (ADR-0008 lines 194-197, 214-217). The haphazardness is that vocabulary-fit is enforced by human review/audit only.
- ADR-0000's two-question reflex is honestly 'review-only and enforced by the faculty it corrupts' (lines 290-295) — no mechanism reads intent, so the ACT of asking 'what type forecloses this class' is unpoliced. The harness cannot force the question but CAN require a defect-fix to NAME its foreclosed class (Revisit #3, lines 364-368).
- Provenance is non-attributable in the broad project: the assignment's framing (non-attributable git-hash-vs-perf, no bug→session link) — the exp_db ledger solves this ONLY for throughput-lab readings/findings; there is no general ledger linking an arbitrary git-hash to the Claude Code session, hypothesis, and interpretation that produced it across chocofarm.
- The closed vocabularies are scattered as inline CHECK strings and Python EnumSets per-file (driver, status x3 tables, git_tree, audit band/severity critical/major/minor, sound/messy) — there is no single queryable REGISTRY of the corpus's closed vocabularies, so 'is this value a member?' and 'does any element fit?' (the ADR-0008 fit test) cannot be asked uniformly.
- ADR supersession / provisional-strength is represented only in PROSE: ADR-0000 is 'Provisional (emergency)' with an unresolved genre tension, ADR-0008 carries an inline '(Amended 2026-06-15: ...)' that supersedes its own example — there is no machine-readable defeasible store of which ADR/amendment supersedes which, mirroring what tlab_finding.supersedes already does for findings.
- The 'deliberately-imprecise tag' / 'cited-not-rerun' / STALE markers (ADR-0008 lines 161-168) are an explicit third truth-value ('unknown/suspect, refusing to classify') that no classical boolean schema can hold without collapsing it to true/false — a paraconsistent/3-valued representation gap.

## Harness hooks (where a registry / ledger / logic-gate plugs in)

- CLASSICAL (Datalog/ASP) — Enum-coverage gate: ingest every closed vocabulary (driver, status enums, git_tree, audit bands) into a registry table; a Datalog/ASP rule flags any literal classification value with no membership and any vocabulary element that descriptively fits nothing (ADR-0008 Rule 1 + fabricated-parent detector, the exact mechanization trigger ADR-0008 Revisit #4 names).
- CLASSICAL (SMT/interval logic) — Partition validator: lift the Criterion dataclass partition check (no gaps, no overlaps over a metric's value-line) and FeatureLayout's contiguous-partition assertion into a reusable Z3/OR-Tools interval-coverage check — provable, not statistical; this is the harness's flagship 'provable beats hunch' instance and directly generalizes exp_db.py's prereg criterion.
- CLASSICAL (Datalog) — Fossil/derivable-field detector: a rule that flags any stored field re-derivable from live geometry (the instance.json overlaps/delta_treasures fossil pattern, ADR-0008 lines 56-69) — 'this column is a function of those, therefore it must not be independently stored.'
- NON-CLASSICAL (defeasible/non-monotonic) — Supersession store: generalize tlab_finding.supersedes (exp_db.py 256-258) into a corpus-wide append-only supersede graph covering ADR amendments and provisional decisions; 'current belief on scope S = the node nothing supersedes' is the non-monotonic query. ADR-0000's provisional status and ADR-0008's inline amendment are the first rows.
- NON-CLASSICAL (temporal/LTL) — Criterion-before-result ordering: the prereg invariant (criterion.git_commit MUST temporally precede the result it judges; one experiment one immutable criterion) is a temporal-logic gate already half-built in tlab_prereg (exp_db.py 268-302) — generalize to 'hypothesis declared before, interpretation linked after' for any session.
- NON-CLASSICAL (paraconsistent / 3-valued) — Provisional/suspect tag layer: represent the 'deliberately-imprecise', 'cited-not-rerun', STALE, 'detector-coupled-and-suspect' tags (ADR-0008 161-168) as an explicit third value so conflicting advisories and not-yet-corroborated 'provisional' findings (status enum) coexist without forcing true/false — the logic gate must NOT explode on contradiction.
- PROVENANCE LEDGER — generalize exp_db.py from throughput-lab-specific to a corpus capability/provenance registry: add the Claude Code session id alongside git_commit on readings/findings (closing the assignment's bug→session attribution gap), and a motivated_change link (already present at exp_db.py line 259-260) to close interpretation→code-change loop.
- REGISTRY ingest of ADR-0011 Rule 1 declarations: parse each ADR rule's named enforcement surface into a table so the harness can query 'which invariants are still review-only?' (presumptively decaying per line 77) and surface the standing mechanization backlog automatically.
