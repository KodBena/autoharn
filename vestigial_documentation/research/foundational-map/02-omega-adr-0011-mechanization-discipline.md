# Report 02 — omega/ADR-0011 Mechanization Discipline (vs chocofarm fork) + meta-review cadence

## Summary

omega's ADR-0011 (docs/adr/0011-mechanization-discipline.md) is the canonical "ADR-0011 safety net": every discipline-stating rule must DECLARE its enforcement surface against a closed 7-rung vocabulary (compile-time / build-CI gate / write-time data constraint / query-time gate / advisory surface / checklist-at-a-named-moment / review-only), and recurrence must convert to a mechanism, not more prose. It frames the maintainer ("the executive") lapse not as laziness but as a structural fact: "policy enforced by one person's attention and memory is structurally weak — only mechanical nets help" (Context); trigger-bookkeeping was "recorded correctly in two ADRs and silently rotted in two others" (Alternatives), and the tenet's whole marginal value is "moving the mechanization assessment from an audit-time observation to a named obligation at corrective-authoring time." omega and chocofarm have FORKED the discipline: omega is native (substrate = 2026-06-01 RCA + 2026-06-10 history audit, 5 rules, "ninth tenet"), chocofarm is "Transferred from the LengYue ADR corpus" (4 rules, "eighth tenet") and re-derives the mechanism instances. Crucially, the provenance/accountability ledger the harness wants ALREADY EXISTS in skeletal form in BOTH forks — omega's audit_log table (commit_sha + actor=application_name session attribution) and chocofarm's 2026-06-24 amendments (code_stamp.py git-hash stamping + tlab_finding Witness belief-chain) — but they are unlinked, fork-divergent, and self-admittedly un-mechanized at the meta level (chocofarm: "no CI sweep"; omega: only a pre-merge-checklist line + two audit-sweep absence-checks). RFC-0001 + the 2026-06-10 corpus audit supply the meta-review cadence (compliance every 6mo, validity annually, delta-only after baseline) that is itself the strongest harness candidate: it is the recurring sweep that would detect an undeclared enforcement surface or an unattributed result.

## Gaps (where the haphazardness lives)

- FORK DIVERGENCE of the enforcement vocabulary itself: omega has 7 rungs incl. query-time-gate/advisory/checklist; chocofarm has 5 rungs incl. construction/import-time + run-time invariant. There is no shared registry of the vocabulary, so the closed-vocabulary discipline (ADR-0008) is violated across the fork boundary — the harness's registry should host ONE vocabulary both projects map onto.
- Provenance is fork-local and UNLINKED: omega stamps writes with commit_sha+actor in audit_log; chocofarm stamps readings with git-hash and findings with commit-formed-against — but there is no cross-project ledger linking git-hash -> benchmark reading -> finding/hypothesis -> the Claude Code session that produced it. The session actor exists only as PGAPPNAME free-text in omega and is absent in chocofarm.
- No bug->introducing-session link anywhere: audit_log records commit_sha on ship-closures but nothing connects a later-discovered defect back to the session/commit that introduced it (the harness's stated missing capability).
- ADR-0011 is self-admittedly un-mechanized at the meta level: chocofarm states 'no CI sweep that detects a discipline-stating rule lacking an enforcement declaration; the architectural audit is that detector, run on demand' (lines 143-149); omega's only nets are a checklist line + audit-sweep absence-checks — both human-attention-gated. The absence-detector for undeclared enforcement surfaces is exactly a logic-gate the harness should provide.
- commit_sha in omega's audit_log is honest-nullable and only set by writers who manually set the audit.commit GUC — so attribution is opt-in and silently absent for non-ship writes; nothing enforces that a result-bearing write carries its commit.
- RFC-0001 is still Draft (line 4) and the docs/audits/ ledger destination it proposes is unbuilt; the meta-review cadence exists as prose, not as a scheduled/queryable mechanism — itself the prose-decays hazard ADR-0011 warns about.
- Operations-research/automated-reasoning capability is invisible: chocofarm has Z3/OR-Tools-class reasoning (equivalence tests, partition assertions, the audit's provable deflations like realizable_static=0.08553) but no machine-readable registry tells an agent these provable tools exist; agents default to statistical hunches.
- The measure-first baseline records (Rule 3) live in scattered prose homes (eslint.config.js rationale header, audit §5, ABS_TOL comments) with no queryable store of 'mechanism X adopted against baseline Y on date Z' — the adr-effectiveness-audits absence-check has nothing structured to query.

## Harness hooks (where a registry / ledger / logic-gate plugs in)

- CAPABILITY REGISTRY should host the SINGLE canonical Rule-1 enforcement-surface vocabulary (union/reconciliation of omega's 7 rungs and chocofarm's 5) as a logic program, with each project's local rungs mapped to it — turning the closed-vocabulary discipline into a cross-fork queryable fact table.
- PROVENANCE LEDGER should generalize omega's audit_log schema (schema.sql lines 159-169) into the shared ledger: keep actor + commit_sha, ADD session-id (Claude Code session) and link rows to chocofarm's tlab_reading/tlab_finding so git-hash -> reading -> hypothesis -> session is one join.
- Adopt chocofarm's tlab_finding model (exp_db.py: status enum provisional/confirmed/retracted + supersedes chain) as the harness's hypothesis/interpretation table; it already separates measurement-fact from interpretation-belief, the structural unconfusability the harness wants.
- Build the absence-detector ADR-0011 admits it lacks: a logic gate over the ADR corpus that fails when a discipline-stating rule has no declared enforcement surface, or a mechanism adoption has no measured-baseline record (the two adr-effectiveness-audits absence-checks, lines 130-145, mechanized instead of checklist-gated).
- Encode work_status_violations (schema.sql lines 105-128) as the template for harness logic-gates — port 'shipped-without-ship-ref' into 'result-without-commit-stamp' and 'finding-without-supersede-on-retract' gates.
- Turn RFC-0001's per-ADR Automatable/Semi/Manual classification (lines 159-187) and the delta-only cadence into a scheduled, queryable meta-review job — the registry knows which ADRs are mechanizable and the ledger records each pass; this is the ADR-0011 safety net as a cron-able mechanism, not a Draft RFC.
- Lift RFC-0001 open-question-9's detection heuristic (lines 308-321: 'for any state X read by 2+ modules, find paths that mutate X without invoking the owner's methods') as a concrete multi-writer logic-gate — directly aligned with the hack-rationalization-detector's 'more than one writer' trigger.
- Encode Rule-4 anti-patterns (omega lines 108-117; chocofarm `it+1_000_000` lines 133-137) as a class of 'enumeration-that-fails-open-at-next-instance' lints the registry can quantify over.
- Register chocofarm's OR/automated-reasoning tools (Z3/OR-Tools-class, the equivalence tests, partition assertions) in the capability registry with their PROVABLE-result guarantee so agents prefer them over StackExchange statistical hunches.
