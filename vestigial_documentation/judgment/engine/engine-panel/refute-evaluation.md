## 2. The pilot-light atom does not close the gap it is introduced to close

- In clingo, a typo'd/undefined predicate in a rule **body** does not abort grounding — it emits an *info/warning* to stderr, the rule simply never fires, `Result=SATISFIABLE`, and every *other* rule still derives. So the mechanism guards exactly the already-guarded case and passes exactly the case named as its motivation.
