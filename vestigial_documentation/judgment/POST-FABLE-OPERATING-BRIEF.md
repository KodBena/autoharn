## Pre-banked judgment — APPLY, NEVER WEAKEN

Odd-link judgment for the standing work already exists on disk. A stand-in serving an
odd link applies it; weakening it, re-deriving it, or "improving" it without a
maintainer ruling is the latitude failure this whole file exists to prevent.

**1. The fixpoint protocol speaks in three knobs, named apart — never one letter.**

- **confirmation-depth** — consecutive clean criterion-review rounds required to
  terminate the loop (the design note's "K"; consult 39 §1 proposes default 1 at
  e-series grain, 2+ reserved for thin in-run review or high stakes).
- **panel-width** — parallel first-contact criterion reviewers within one round
  (consult 37 §2's "K=2 criterion-reviews, lens-split"; e18 ran width 2, depth 1).
- **round-ceiling** — the mandatory hard cap. Cost is a real constraint; an unbounded
  polish loop is a budget hazard, not a virtue. Cap-hit = the unit closes RED-honest
  with its open findings, never auto-attested.

The single letter K carried the first two meanings simultaneously in two live documents
(`claude_harness/docs/design-notes/review-fixpoint-protocol.md` vs consult 37 §2) — the
maintainer caught the drift; the specimen is in the catalog below. When reading either
document, or consult 39's "K drops to 1", translate into the three-knob vocabulary
before acting; when writing, the bare letter is banned.

Fixpoint application rules, unchanged from the design note + consult 39: only a
**criterion-review** (fresh first-contact stamp, briefed on artifact + spec + fixtures
ONLY — no fix narrative, no prior verdicts, no round number) terminates the loop; a

structural criterion (zero findings surviving actor-attributed disposition), never the
verbal verdict "flawless"; and criterion reviewers MUST be subagents — a fresh session's
main context stamps as "main" and collides with the author under agent-granular
distinctness (e18 mechanism fact, recorded as operational law pending ratification).

## The verification discipline (ADR-0014, operationalized)

NEVER adjudicate a completion report from its text. Every acceptance this stretch was
preceded by source-level spot-checks, and confidently-repeated claims have been FALSE
while asserted. The checklist that caught everything:

1. The claimed commits exist, in both repos, with plausible diffs (`git show --stat`).
2. Every load-bearing claim grep-verified at source (the fix is in the file, the gate
   is in the trigger list, the deleted mode has zero residual references).
3. Test suites re-run independently — never trust a reported count.
4. Byte-identity claims checked via git (did the banked artifacts' files change?).
5. Live-state claims read directly (psql read-only) — a report's description of DB
   state is a claim, not an observation.
6. **Citation currency**: any citation of repo or DB state — including your own prior
   documents' — is re-observed at the moment of citation. Cited-from-memory is the one
   place an otherwise-faithful author is reliably wrong (flaw-1 specimen below).
7. **Provenance gravity check**: before adopting any prior document's position, check
   the deliberations and the rulings stream for an act that superseded it.

## The maintainer interface

He is non-technical and executive-level, and the interface must be built for that,
deliberately:

- **Briefs to him are plain-language option sets with a recommendation.** Name the
  options, the cost of each, and one recommended choice with its reason — then let him
  decide. Never a wall of mechanism; never jargon that makes his ratification a
  formality because he could not evaluate it. A leading question with the conclusion
  pre-loaded is the ADR-0013 Rule 3 violation, not a service.
- **Pastes destined for the vicar (or any relay) must be complete and self-contained.**
  He pastes verbatim; he cannot repair a fragment. A paste that assumes context the
  target does not have is a broken instrument.
- **Never leave him guessing where his fingers go.** Every step he must perform names
  the exact command, file, or button, in order, ready to execute. The arm-script idiom
  is the template: everything checkable is checked mechanically, and what remains for
  him is printed as an ordered checklist.
- **Cost is a hard constraint.** Never casually recommend paid services or expensive
  runs; surface cost in every option set where it applies.

## Operational idioms (mechanics that must not drift)

- **The anchor idiom.** Compute the sha256 CLIENT-SIDE over the verbatim STRING you are
  filing (hashlib over the exact bytes), then INSERT via psql with the payload on
  stdin (heredoc/`-f -`), never `-c` — shell interpolation mangles the verbatim and the
  stored `verbatim_sha256` stops matching what a reader recomputes. The row is only an
  anchor if an independent reader can re-derive its hash from the named file.
- **The staging guard (finding 33, fc20).** Every commit declares its intended paths via
  `CLAUDE_COMMIT_PATHS` (exported before `git commit`); the pre-commit guard refuses an
  undeclared or over-scoped staging. Stage explicitly; never `git add -A`.
- **Sonnet relays for dense psql.** Long query batteries and trailer-fact verification
  go through a Sonnet relay agent; the odd link reads the relayed results and treats
  them per checklist item 5 (a relay's description is a claim — spot-check the
  load-bearing rows yourself).
- **The arming automation.** The arm script (template: `harness/e18-build/arm_e18.sh`,
  fc25) renders the arming checklist CHECKABLE: `--verify` runs every pre-arm check
  mechanically (DDL on a throwaway, frozen-text sha match, hook interceptability,
  fixtures, the ambiguity pre-test's two-consecutive-empty token) and emits the frozen
  delivery set; the maintainer's host-side steps print as an ordered checklist. NOTHING
  arms by itself; arming is the maintainer's act, gated by the checklist.
- **Both-polarity gate discipline.** A gate never seen red is a claim (ADR-0011): every
  gate and close line ships with fixtures proving it green on the honest case AND red on
  the violation (the seen-red evidence lives under
  `claude_harness/docs/adr-evidence/seen-red/`). This includes negative controls on the
  meta-toolchain — a never-invoked close must be distinguishable from a green one.
- **The blind-consult protocol.** For any consult whose value is independence: commit
  your own shape FIRST (commit order proves the independence — the engine SEED's
  provenance section is the worked example), then brief the consulted agents with FACTS
  ONLY — the symptom, the measurements, the artifacts — never your hypothesis, suspect,
  or mechanism. Commit their outputs verbatim; the synthesis cites them, it does not
  replace them. Record every panelist's self-reported model with the standing caveat.
