# Reference-graph SSOT — what a *referent* is and what a *reference* means

This directory is the **project-local single source of truth** for autoharn's reference graph: the
one place that defines the node/edge/annotation vocabulary. Every downstream artifact (a Cytoscape
import, a Graphviz `.dot`, a report) is a **compile target** of a graph *instance* that conforms to
`schema.json` — never an independent hand-authored source (that would be the two-writers failure the
project's own P1/ADR-0005 forbid).

A graph instance is one JSON document: `{ "version", "nodes": [...], "edges": [...] }`.

## Granularity rule (hard boundary)

Nodes exist at exactly two granularities, and **never below file level**:

- **`directory`** — a tracked directory (`gates/`, `kernel/lineage/`). Rendered as a *compound /
  collapsible* parent, so a viewer can zoom from directory-level to file-level and back. This is how
  "different granularities, clickable boxes" is realized (Cytoscape compound nodes / Graphviz
  clusters).
- **`file`** — a single tracked file (`gates/staging_guard.py`, `law/adr/0008-*.md`). A file is the
  atom. Functions, classes, SQL statements, doc sections — **out of scope, by rule.**

A Python module and its file are the *same node* (module ≡ file here); there is no separate
"module" granularity below the file.

## Node

| field | required | meaning |
|---|---|---|
| `id` | yes | repo-relative path, the stable identity (`instruments/close_manifest.py`). |
| `label` | yes | display name (usually the basename). |
| `granularity` | yes | `directory` \| `file`. |
| `kind` | yes | `doc` \| `python` \| `sql` \| `shell` \| `config` \| `dir` \| `other` — what the atom *is*. |
| `parent` | no | the containing `directory` node id, or omitted for a top-level node. Drives compound nesting. |
| `dirclass` | no | `CORE` \| `DOC` \| `RESEARCH` \| `OTHER`, inherited from `DIRCLASS.md` (the classification SSOT). |
| `mode` | no | `use` \| `study` \| `shared` — the Use-mode/Study-mode axis (see `WALKTHROUGH.md`). Interpretive. |
| `vestigial` | no | boolean — legacy/superseded/parked and not load-bearing for a functioning harness. |
| `vestigial_reason` | no | one line; required when `vestigial` is true. |

## Edge — the closed vocabulary of "reference"

An edge is a **directed** reference `source → target`, both node ids. `type` is what the reference
*means* — the discipline is that this vocabulary is closed and each type has a decidable test:

| `type` | source → target | the test (what makes it that reference) |
|---|---|---|
| `imports` | python → python | `source` has an `import`/`from` that resolves to `target` within the repo. |
| `applies` | shell/python/sql → sql | `source` executes/sources `target` DDL (`psql -f`, `\i`, `source`) — the apply/lineage order. |
| `invokes` | shell/python → executable | `source` runs `target` as a subprocess/hook (e.g. `hooks/pre-commit` → `gates/*.py`). |
| `wires` | config → hook/tool | `source` config registers `target` as a live hook/tool (`.claude/settings.json` → `hooks/*.py`). |
| `cites` | doc → any | `source` prose references `target`: a markdown link, `ADR-NNNN`, `finding N`/`FN`, a `[[glossary-term]]`, or `GLOSSARY.md#anchor`. |
| `enforces` | gate/hook/instrument → law | `source` mechanically enforces the discipline `target` states in prose (a gate → the ADR/brief/finding it mechanizes). **Interpretive** — Sonnet-assigned, not regex. |
| `tests` | test/fixture → any | `source` exercises `target` (a `test_*.py` / `*_fixture.py` against a module or behavior). |
| `supersedes` | record → record | `source` supersedes/derives-from an earlier `target` (kernel `sNN` lineage, migration provenance). |

`evidence` (optional, recommended): where the reference lives — `path:line`, or a short note for
interpretive edges. `note` (optional): free text.

## How an instance is produced (two passes)

1. **Deterministic pass** — a mechanical extractor emits all nodes and the *decidable* edge types
   (`imports`, `applies`, `invokes`, `wires`, and the regex-decidable `cites`). No judgment.
2. **Interpretive pass (Sonnet workflow)** — adds the judgment edges (`enforces`, semantic `cites`
   the regex missed, `supersedes`) and the `mode` / `vestigial` node annotations.

Keeping the two passes distinct keeps the honest boundary visible: a mechanical edge is checkable; an
interpretive edge carries a model's judgment and says so in `evidence`.

## Compile targets (not sources)

- **Cytoscape** — `elements: { nodes, edges }`; `parent` → compound boxes; `dirclass`/`type` → CSS
  classes for styling; collapsible parents give the granularity zoom.
- **Graphviz `.dot`** — top-level directories as `subgraph cluster_*`; edge style per `type`.

Both are regenerated by the compiler from one instance; neither is edited by hand.
